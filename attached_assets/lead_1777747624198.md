Build a production-ready AI Lead Calling & Qualification System using Node.js.

## 🎯 Objective
Create a backend system that:
- Accepts leads (manual + CSV upload)
- Calls leads automatically using Twilio
- Streams audio in real-time
- Integrates with Sarvam AI for voice conversation
- Analyzes call and updates lead status
- Maintains logs and dashboard APIs

---

## 🧱 Tech Stack
- Node.js (Express or Fastify)
- PostgreSQL (database)
- Redis + BullMQ (queue system)
- WebSocket (for real-time audio streaming)
- Twilio (voice API + media streams)
- Sarvam AI (voice agent)

---

## 📦 Modules to Build

### 1. Auth Module
- JWT-based authentication
- Login / Register APIs

---

### 2. Lead Module
- Upload leads (CSV + manual API)
- Fields:
  - name
  - phone
  - source
  - status (pending, calling, completed, interested, not_interested, no_response)
- APIs:
  - POST /leads
  - POST /leads/upload
  - GET /leads
  - GET /leads/export

---

### 3. Queue System (BullMQ)
- Queue name: lead-call-queue
- Process leads one-by-one
- Retry logic:
  - max 3 retries
  - delay 2 hours

---

### 4. Call Service (Twilio Integration)
- Use Twilio Programmable Voice API
- Create outbound call:
  - POST /call/initiate
- Twilio webhook:
  - POST /voice
  - return TwiML with <Connect><Stream> WebSocket

---

### 5. WebSocket Server
- Handle Twilio Media Streams
- Receive audio chunks
- Send to Sarvam AI
- Receive AI response audio
- Stream back to Twilio

---

### 6. AI Integration (Sarvam)
- Create service:
  - send audio → get response
  - conversation memory
- Prompt:
  - act as sales agent for CRM
  - qualify lead
  - detect interest
  - ask for demo

---

### 7. Call Status Webhook
- POST /call-status
- Handle events:
  - initiated
  - ringing
  - answered
  - completed
  - no-answer
  - busy
- Update DB

---

### 8. Call Logs Module
- Store:
  - lead_id
  - call_status
  - duration
  - recording_url
  - transcript

---

### 9. AI Analysis Module
- Analyze transcript after call
- Return:
  {
    interest: "high | medium | low",
    next_action: "demo | follow_up | drop"
  }
- Update lead status

---

### 10. Dashboard APIs
- GET /dashboard/stats
- GET /calls
- Filters:
  - interested
  - not answered
  - follow-up

---
